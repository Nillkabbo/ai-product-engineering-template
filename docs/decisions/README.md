# Architecture Decision Records (ADR)

Purpose
Use ADRs to document significant architecture decisions that affect the system over time.

Rules
- One decision per file
- Use the template in template.md
- Number ADRs sequentially (0001, 0002, ...)
- Keep ADRs short and focused

Workflow
1. Copy template.md
2. Assign the next number
3. Fill in context, decision, and consequences
4. Link to related tasks if applicable
